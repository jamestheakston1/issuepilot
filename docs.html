Overview

This comprehensive guide provides details on linking your GitHub account, managing account deletion, and integrating with the IssuePilot Developer API.

Linking

1. Linking Your Primary GitHub Account

This is the standard process for connecting your main GitHub identity to IssuePilot. You must provide consent for IssuePilot to access your public repositories and necessary profile data.

Steps:

  1. Visit IssuePilot
  2. Click the 'Connect with GitHub' Button.
  3. Review permissions if you wish.
  4. Click 'Authorise IssuePilot' at the bottom of the page.

2. Selecting an Account During Authorization

If you are currently signed into multiple GitHub accounts in your browser, the GitHub authorization screen will prompt you to select which account you want to link to IssuePilot.

Steps:

  1. Visit IssuePilot.
  2. Click the 'Connect with GitHub' Button. You will be redirected to the GitHub account selection page.
  3. Select the account you wish to link IssuePilot to and then you will be redirected to the GitHub authorization page.
  4. Review permissions if you wish.
  5. Click 'Authorise IssuePilot' at the bottom of the page.

Only the single account you select on this screen will be linked to your IssuePilot profile.
